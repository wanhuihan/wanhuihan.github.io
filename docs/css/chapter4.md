本章介绍最重要的三种改变文档流的方式，浮动，FlexBox和网格布局。还会介绍定位。
接下来也会展示两种常见的网页布局模式： 双容器模式和媒体对象。

## 浮动的设计初衷
浮动能将一个元素拉到其容器的一侧，这样文档流可以包围它，浮动元素会被移出正常文档流，文档流会重新排列。在之前都是通过浮动来解决布局问题，后来有了display:inline-block和display: table, 随着Flexbox和网格布局的问世，float浮动渐渐用的人比较少了。

## 容器折叠和清除浮动

浮动不会增加父级元素的高度。

清除浮动有个配套的属性clear。将一个元素放在主容器的末尾，并对他用clear，这会让容器扩展到浮动元素下面。clear:both 声明让该元素移动到浮动元素的下面，而不是侧面。 clear还有left, right值，清除左浮动和右浮动。

也可以使用伪类进行清除浮动。
```css 
.clearfix::after {
    clear:both;
    content: "";
    display: block;
}
```
这个清除浮动还有一个问题没解决，浮动元素的外边距不会折叠刀清除浮动容器的外部，非浮动元素的边距则会正常折叠。
解决方案
```css
.clear::before, .clear::after {
    display: table;
    content: ""
}
.clear::after {
    clear:both;
}
```

## 出乎意料的浮动陷阱

三个左浮动的盒子，盒子1比盒子2高，则盒子3不会浮动到左边。

解决方案：
将每行第一个元素添加伪类清除左浮动或者右浮动
```css
/** odd是基数，even是偶数，如果每行的浮动元素不定，则需要使用 n + 1 等 */
.media:nth-child(odd) {
    clear: left; /* 如果是左浮动的话 */
}
```
