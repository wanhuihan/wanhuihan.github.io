浮动的设计初衷 #

浮动能将一个元素拉到其容器的一侧，这样文档流可以包围它，浮动元素会被移出正常文档流，文档流会重新排列。在之前都是通过浮动来解决布局问题，后来有了display:inline-block和display: table, 随着Flexbox和网格布局的问世，float浮动渐渐用的人比较少了。

容器折叠和清除浮动 #

浮动不会增加父级元素的高度。

清除浮动有个配套的属性clear。将一个元素放在主容器的末尾，并对他用clear，这会让容器扩展到浮动元素下面。clear:both 声明让该元素移动到浮动元素的下面，而不是侧面。 clear还有left, right值，清除左浮动和右浮动。

也可以使用伪类进行清除浮动。

css
.clearfix::after {
    clear:both;
    content: "";
    display: block;
}

出乎意料的浮动陷阱 #

三个左浮动的盒子，盒子1比盒子2高，则盒子3不会浮动到左边。

解决方案： 将每行第一个元素添加伪类清除左浮动或者右浮动

css
/** odd是基数，even是偶数，如果每行的浮动元素不定，则需要使用 n + 1 等 */
.media:nth-child(odd) {
    clear: left; /* 如果是左浮动的话 */
}

媒体对象和BFC #

BFC(block formatting context)块级格式化上下文

图片进行了左浮动，文字会自动靠右排列了，我所希望的是左右结构，文字不超出，这就需要使用到BFC。 强制给一个元素生成一个新的BFC, 他不会跟其他BFC重叠。

给元素生成一个新的BFC， 它不会跟其他BFC重叠。

给元素添加一下人艺术性质都会创建BFC。

css
float: /* left or right, not none */
overflow: /* hidden, auto or scroll, not visible */
display: /* inline-block,table-cell, table-caption,flex, inline-flex, gird or inline-grid. */
position: /** absolute or fixed */

网格系统 #

网格系统可以提高代码的复用性。大部分css框架都自带网格布局，比如bootstrap, Fundation以及Pure.

网格列的css 12可以被2，3，4，6整除，通常以12列为准。

css
[class*="column-"] {
    float: left;
}
.column-1 {
    width: 8.3333%
}
.column-2 {
    width: 16.6667%;
}
.column-3 {
    width:25%
