  display: grid;

与FlexBox的配合 #

Flexbox本质是一维的，而网格是二维的。 Flexbox是以内容为切入点由内向外工作，而网格是以布局为切入点从外向内工作的。

替代语法 #

布局网格元素还有另外两个替代语法，命名的网格线和命名的网格区域

命名的网格线 #

css
grid-template-columns: [start] 2fr [center] 1fr [end];
1

这条声明定义了两列的网格，三条垂直的网格线分别叫做start, center, end。

css
.a {
    grid--template-columns: [left-start] 2fr [left-end right-start] 1fr [right-end];
}
1
2
3

这条声明里，2号网格线既叫left-end,也叫right-start，之后可以任选一个名称使用。

命名网格区域 #

另一个方式是明明网格区域。不用计算或明明网格线，直接用明明的网格区域将元素定位到网格中。 实现这一方法需要借助网格容器的grid-template属性和网格元素的gird-area属性。 gird-template-areas属性使用了一种SCII art语法，可以直接在css中画一个可视化的网格形象；

css
.a {
    grid-template-areas: "top  top    right"
                         "left .      right"
                         "left bottom bottom"
}
1
2
3
4
5

可以用.作为名称，这样便能空出一个网格单元。

如果是明确知道每个网格元素的位置，命名网格区域语法比较合适。 网格布局共设计了三种语法：编号的网格线、命名的网格线、命名的网格区域。

显式和隐式风格 #

在某些场景下，你可以不清楚把该元素放在网格的哪个位置上。当处理大量的网格元素时，挨个制定元素的位置未免太不方便。这个时候，以一种宽松的方式定义网格可能更合理，剩下的交给布局算法来放置网格元素。

这时需要用到隐式网格（implicit grid）。使用grid-template-*属性定义网格轨道时， 创建的是显式网格（explicit grid)。 隐式网格轨道默认大小为auto.
